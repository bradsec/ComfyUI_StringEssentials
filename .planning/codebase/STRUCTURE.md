## Naming Conventions

**Files:**
- Node files: `string_<operation>_node.py` format
  - Examples: `string_strip_node.py`, `string_multi_replace_node.py`
  - Pattern: snake_case with `_node.py` suffix

**Classes:**
- Node classes: PascalCase with "Node" suffix
  - Examples: `StringTextboxNode`, `StringStripNode`, `StringMultiReplaceNode`
  - Pattern: Convert filename from snake_case to PascalCase, append "Node"

**Methods:**
- Execution methods: verb_noun_v1 format (snake_case)
  - Examples: `string_strip()`, `string_replace()`, `conditional_append()`, `contains_any()`, `pass_string()`, `preview_string()`
  - Pattern: Descriptive snake_case names indicating operation

**Variables:**
- Instance variables: snake_case with descriptive names
  - Examples: `input_string`, `strings_to_remove`, `match_case`, `preserve_punctuation`
  - Pattern: Clear names indicating type and purpose

**Constants:**
- Node metadata: UPPERCASE
  - Examples: `RETURN_TYPES`, `RETURN_NAMES`, `FUNCTION`, `CATEGORY`, `INPUT_TYPES`, `OUTPUT_NODE`
  - Pattern: ComfyUI convention for node interface

## Where to Add New Code

**New String Operation Node:**
1. Create file: `string_<operation>_node.py` in root directory
2. Class structure (follow pattern from existing nodes):
   - Define `INPUT_TYPES()` classmethod with parameter definitions
   - Define `RETURN_TYPES` tuple with output types
   - Define `RETURN_NAMES` tuple with output names
   - Set `FUNCTION = "<method_name>"`
   - Set `CATEGORY = "utils/StringEssentials"`
   - Implement execution method with logic
3. Add to `__init__.py`:
   - Import the new class at top
   - Add entry to `NODE_CLASS_MAPPINGS` dict (key = class name as string)
   - Add entry to `NODE_DISPLAY_NAME_MAPPINGS` dict (key = class name, value = display name)
4. Add tests to `test_string_nodes.py`:
   - Create test class or add methods to `TestStringNodes`
   - Test basic functionality, edge cases, parameter variations

## File Organization Rationale

**Flat root structure for nodes:** Each node is a separate file at root level
- Rationale: Simple discovery by ComfyUI, clear 1:1 mapping between file and node class
- Alternative avoided: Subdirectory organization (would complicate imports in `__init__.py`)

**Single test file:** All tests in `test_string_nodes.py`
- Rationale: Small codebase, all nodes similar in nature, easy to run all tests together
- Scalability: If adding more nodes, consider splitting tests by node when >100 test methods

**Shared patterns duplicated:** `cleanup_text()` method exists in both `string_strip_node.py` and `string_multi_replace_node.py`
- Current state: Intentionally duplicated to keep each node self-contained
- Future improvement: Extract to shared module if >3 nodes need it
